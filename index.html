A CLI · bh

Skip the dashboard. Pipe it.

Every action in the web app maps to a bh subcommand. Scriptable, idempotent, JSON-out. Built for the kind of person who deploys 30 staging boxes and tears them down before lunch.

  • 01bh servers new — spin up a server in one command.
  • 02bh user — what's left in your account.
  • 03bh regions — list all available regions.
  • 04bh servers delete — tear down a server when you're done.

B REST API

Pipe servers into existence from any language.

Bearer-token auth, JSON in / JSON out. Deploy, resize, snapshot, and tear down programmatically — the same primitives the dashboard and the bh CLI use under the hood.

  • post/servers — provision a new VM across any provider.
  • get/servers/:id — full state, including IP & billing meter.
  • post/servers/:id/resize — bump size, reboot once.
  • del/servers/:id — stop billing immediately.

    Yes, you must use a valid email. We will send you reminders if your account balance is too low before your server is removed. Users who use disposable e-mails will be banned without any notification.

    Why do you have a server limit with Bithost?

    By default, all accounts are limited on how many servers they can deploy. If you would like this limit increased, then please just submit a support ticket.

    Is it secure?

    We do not store any card details, so you do not need to worry about them being leaked. You can also protect your account with 2-step verification. Learn how to enable it →

    Why is it more expensive than DigitalOcean, Linode, and Vultr?

    This is to mitigate the risk of converting cryptocurrencies back to USD, cover all bank taxes etc, and to help with the running costs of bithost.
